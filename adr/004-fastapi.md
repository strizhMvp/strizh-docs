# ADR-004: FastAPI вместо Django / Flask

**Статус:** Принято  
**Дата:** 2026-05-04

## Контекст

Выбор Python web-фреймворка для всех 4 backend-сервисов.

## Решение

FastAPI для всех сервисов.

## Обоснование

**Django:** sync-first, монолитный по природе. DRF добавляет ещё один слой сериализации поверх Pydantic — избыточно для микросервиса. ORM ориентирован на SQL.

**Flask:** нет async из коробки, нет автогенерации OpenAPI, нет встроенной Pydantic-интеграции.

**FastAPI:**
- Async-first — необходим для SSE-стриминга LLM и параллельных I/O вызовов
- Автогенерация OpenAPI — Андрей получает `/docs` и `/openapi.json` каждого сервиса без дополнительных усилий
- Pydantic v2 встроен — одни модели используются для валидации входа, ответов и Beanie-документов
- `sse-starlette` нативно интегрируется через `EventSourceResponse`

## Последствия

- Единый стек во всех 4 сервисах — меньше контекстного переключения
- Андрей может смотреть живую документацию на `/docs` каждого запущенного сервиса
